# Flexible Daily Schedule — Design

**Date:** 2026-06-20
**Scope:** Replace the ephemeral "Plan my day" flow with a persistent, fully editable per-day schedule rendered as a timeline, with the day's Outlook events woven in.

## Problem

Today the daily schedule is generated by the LLM on every request and is throwaway — the only way to keep it is to push each block into Outlook. There is no persistence and no way to tweak the plan. The user wants the generated plan to land directly into the day's schedule, stay editable forever (add / edit / delete slots), and treat pushing to Outlook as an optional, on-demand action rather than the only outcome.

## Decisions (locked during brainstorming)

- **Interaction model:** timeline / agenda grid (vertical time axis) with the day's real Outlook events woven in read-only.
- **Editing:** tap-to-edit (click a block → popover for title / start / duration / notes; click empty timeline → add a slot at that time). No drag-and-drop in v1.
- **Persistence:** per-day, revisitable for any day (not just today).
- **Enhancements in scope:** mark-done + day progress; conflict/overlap highlighting; fill-the-gaps regenerate; selective push to Outlook; now-line + current-block highlight.
- **Out of scope (YAGNI):** natural-language quick-add, carry-over of unfinished blocks, day templates, drag-and-drop.

## Data model

New table **`schedule_blocks`** (SQLAlchemy model `ScheduleBlock`, auto-created on startup via `Base.metadata.create_all` — no Alembic, consistent with `email_preferences`).

| column | type | notes |
|---|---|---|
| `id` | UUID pk | |
| `user_id` | UUID FK→users.id (ondelete CASCADE), indexed | ownership |
| `day` | Date | the day this block belongs to |
| `start_minute` | Integer | minutes from local midnight (540 = 9:00) |
| `duration_minutes` | Integer | |
| `title` | String(500) | |
| `notes` | Text, nullable | |
| `category` | String(20) | `fixed_event`/`task`/`rule_based`/`suggested`/`manual` |
| `importance` | Integer, nullable | from AI |
| `done` | Boolean, default False | |
| `locked` | Boolean, default False | protected from `fill_gaps` regeneration |
| `outlook_event_id` | String(255), nullable | set once pushed → dedupe |
| `source` | String(10) | `ai` / `manual` |
| `created_at` | DateTime(tz) | |

Storing minutes-from-midnight plus a `day` the browser supplies means the **frontend owns "what day / what time is now,"** sidestepping the prior UTC "today" drift.

## API

New router **`src/api/schedule.py`** (prefix `/schedule`), registered in `src/main.py`. A shared generator service **`src/services/schedule_ai.py`** holds `generate_blocks(user, db, day, intent, keep_blocks, events) -> list[block dict]`, refactored out of the current `daily_schedule.py` LLM logic. The AI prompt is updated to require a concrete clock start time for **every** block (no "Flexible"/"Evening") since blocks now sit on a timeline.

| method · path | behavior |
|---|---|
| `GET /schedule?date=YYYY-MM-DD` | `{ day, blocks[] (saved, ordered by start_minute), events[] (that day's Outlook events, read-only) }` |
| `POST /schedule/block` | create `{date, start_minute, duration_minutes, title, notes?}` → `source=manual` |
| `PATCH /schedule/block/{id}` | partial update of title / start_minute / duration_minutes / notes / done / locked |
| `DELETE /schedule/block/{id}` | delete |
| `POST /schedule/generate` | `{date, intent?, mode:"replace"\|"fill_gaps"}`; persists AI blocks. `replace` = delete the day's non-locked blocks then write fresh AI blocks; `fill_gaps` = keep all existing blocks + Outlook events, generate only into empty time around them |
| `POST /schedule/block/{id}/push` | create Outlook event via `OutlookCalendarService.create_event`, store `outlook_event_id`; no-op if already set |
| `POST /schedule/push?date=` | push all un-pushed blocks for the day |

All endpoints are `current_user`-scoped with per-row ownership checks (matching the data-isolation pattern). The legacy `GET /daily-schedule` ephemeral flow and its frontend ("Plan my day" card, `planDay`/`pushPlan`) are retired. `create-task-from-block` is removed unless a use remains (decided in the plan).

Conflict detection, the now-line, and the progress bar are computed **on the frontend** — no endpoints.

## Frontend (`src/static/index.html`)

Inside the day-detail (`selectDay`), the read-only "Schedule" list and the "Plan my day" card are replaced by the timeline component:

- **Control bar:** intent input; **Generate day** (mode=replace), **Fill the gaps** (mode=fill_gaps), **Push all to Outlook**, **Add slot**; a "X of Y done" progress bar.
- **Timeline:** left time axis; the canvas height maps a dynamic time window (min of earliest block/event or a default like 7:00, to max of latest end or ~22:00). Your blocks solid (accent), Outlook events dashed/hatched read-only, positioned by `start_minute`/`duration`.
- **Per-block:** done checkbox (done → greyed + struck), a 🔒 lock toggle, tap → edit popover (title / start / minutes / notes; Save · Delete · → Outlook).
- **Now-line:** red line at the current local time (today only), current block highlighted.
- **Conflicts:** any two of your blocks that overlap, or a block overlapping an Outlook event, get a red outline + a short ⚠ note.
- Reuses Hand-Drawn CSS variables; all fetches go through the existing `calJson()` helper and `withBusy()` button guards.

## Error handling

- Generate/push failures surface via the existing status helpers; partial push reports created/failed counts.
- Block writes are ownership-checked; a missing/foreign block id returns 404.
- `localStorage`/session-expiry handling already covered by `calJson()`.

## Testing (stdlib `unittest`)

- `schedule_ai`: time-string → `start_minute` parsing; `fill_gaps` excludes time ranges already covered by kept blocks + events.
- API: create/patch/delete ownership enforcement; `generate` replace vs fill_gaps block counts (LLM mocked); `push` sets `outlook_event_id` and is idempotent (mocked `create_event`).

## Out of scope

Drag-and-drop, natural-language add, carry-over, templates, server-side conflict computation.
